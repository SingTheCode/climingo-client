# 개발 워크플로우 체크리스트

## 새로운 기능 개발

### 1. 설계 단계
- [ ] 요구사항 분석 완료
- [ ] 아키텍처 레이어 결정 (Controller/Service/State/Repository/Transform)
- [ ] 엔티티 타입 정의 (`types/[domain].d.ts`)
- [ ] API 명세 확인
- [ ] 컴포넌트 구조 설계

### 2. 구현 단계
- [ ] Transform 함수 작성 (`stores/transforms/[domain].ts`)
  - [ ] 옵셔널 체이닝 + Nullish Coalescing 사용
  - [ ] 완전한 엔티티 객체 반환 보장
- [ ] Repository 구현 (`stores/api/[domain].ts`)
  - [ ] Transform 함수 import
  - [ ] 순수 API 통신만 담당
- [ ] State 구현 (`stores/modules/[domain].ts`)
  - [ ] 상태 정의
  - [ ] API 호출 액션 작성
  - [ ] 로딩 상태 관리
- [ ] Service 구현 (필요시, `hooks/[domain].ts`)
  - [ ] 순수 비즈니스 로직만
  - [ ] Store/ref/computed 사용 금지
- [ ] Controller 구현 (`components/[domain]/`)
  - [ ] Store 접근
  - [ ] computed/ref 선언
  - [ ] UI 로직 구현

### 3. 테스트 단계
- [ ] 단위 테스트 작성
  - [ ] Transform 함수 테스트
  - [ ] Service 로직 테스트 (있는 경우)
- [ ] 통합 테스트 작성
  - [ ] Repository 테스트
  - [ ] State 테스트
  - [ ] 컴포넌트 테스트
- [ ] E2E 테스트 작성 (주요 플로우)
- [ ] 모든 테스트 통과 확인 (`npm test`)

- [ ] 민감 정보 하드코딩 없음

### 5. 코드 품질
